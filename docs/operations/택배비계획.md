# 택배비계획

## 📦 택배비 산정 규칙 — 전체 분석

현재 시스템에 **2개의 택배비 계산 체계**가 공존합니다.

---

## 🔷 체계 1: `ShippingCalculator.php` (메인 SSOT)

> 파일: `includes/ShippingCalculator.php` (1,261줄)
> 문서: `docs/features/shipping.md`

**핵심 원리**: `무게(kg) → 20kg 기준 박스 분리 → 택배비 산정`

### 1. 무게 계산 공식 (품목별)

| 품목 | 무게 공식 | 비고 |
|------|-----------|------|
| **전단지** | `평량(gsm) × 면적(m²) × 매수 × 코팅가산` | 코팅: 유광/무광 ×1.04, 라미 ×1.12 |
| **봉투** | `gsm × 펼침면적(m²) × 수량` | 대봉투 510×387, 소봉투 238×262 |
| **스티커** | `(재질gsm + 후지120gsm) × (가로mm/1000 × 세로mm/1000) × 수량` | 후지(이형지) 120g 고정 가산 |
| **NCR양식지** | `gsm × 면적(m²) × 총매수` | NCR지=60g, 일반양식지=모조gsm |
| **명함/상품권** | `3g/매 × 수량` (간이) | 소형 제품 |
| **자석스티커** | `15g/매 × 수량` (간이) | 소형 제품 |

### 2. 박스 분리 기준

## 🔗 택배 선불 고객 적용 프로세스

```
고객 주문 시
  └─ 물품수령방법: "택배" 선택
     └─ 운임구분: "선불" 라디오 선택
        └─ AJAX → ShippingCalculator::estimateFromCart() → 추정 무게만 표시
           └─ ⚠️ "추정치이며 실제와 다를 수 있습니다" + ☎ 전화 안내
              └─ 주문 완료 (택배비 미확정 상태)

관리자
  └─ 주문 확인 → 실제 무게 측정 / 택배사 확인
     └─ shipping_api.php?action=logen_save → logen_delivery_fee 저장
        └─ 고객에게 이메일 자동 발송 (택배비 안내 + 마이페이지 결제 링크)

고객 결제
  └─ 마이페이지 → 주문상세 → 결제하기
     └─ 인쇄비(money_5) + 택배비(logen_delivery_fee) + VAT 합산 결제
```

---

## ⚠️ 핵심 포인트

1. **택배비는 "추정"** — ShippingCalculator는 참고용 추정치만 제공, 최종 확정은 관리자 수동
2. **2중 체계** — ShippingCalculator(무게 기반, 정밀)와 delivery_rules_config(수량 기반, 간이)가 공존
3. **선불 고객 택배비는 주문 시 확정 안 됨** — 관리자가 확정 후 고객이 추가 결제
4. **DB 테이블**: `shipping_rates` (요금표), `mlangorder_printauto.logen_delivery_fee` (확정 택배비)
