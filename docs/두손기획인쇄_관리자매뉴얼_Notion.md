웹 브라우저에서 다음 주소로 접속합니다:

관리자 계정으로 로그인하면 대시보드 메인 화면이 표시됩니다.

### 1.2 대시보드 메인 화면

## 2. 주문·교정 관리

### 2.1 관리자 주문 등록

전화/비회원 주문을 관리자가 직접 등록합니다.

1. 사이드바 > 주문·교정 > 관리자 주문 클릭
2. 품목 선택 → 카테고리/수량/사이즈 입력
3. 주문자 정보 (이름, 전화, 이메일, 주소) 입력
4. 가격 수동 입력 (공급가액 + VAT 자동 계산)
5. 배송방법/결제방법 선택 → 주문 등록

> **TIP** 택배 선불 선택 시 택배비 금액을 직접 입력할 수 있습니다.

### 2.2 주문 목록 및 상태 관리

모든 주문을 한 화면에서 확인하고 상태를 변경합니다.

**주문 상태 (OrderStyle)**

| 코드 | 상태명 | 설명 |
| --- | --- | --- |
| 0 | 접수대기 | 고객이 주문을 제출한 상태 |
| 1 | 입금대기 | 결제 대기 중 |
| 2 | 입금확인 | 결제 확인 완료 |
| 3 | 접수완료 | 관리자가 접수 처리 |
| 4 | 시안확인중 | 교정 시안 확인 단계 |
| 5 | 인쇄준비 | 인쇄 준비 중 |
| 6 | 인쇄중 | 인쇄 진행 중 |
| 7 | 후가공 | 후처리 단계 |
| 8 | 작업완료 | 제작 완료 |
| 9 | 발송완료 | 고객에게 발송 완료 |
| 10 | 수령완료/반품 | 고객 수령 또는 반품 |

목록에서 드롭다운으로 직접 상태 변경이 가능합니다 (인라인 상태 변경).

### 2.3 주문 상세 화면

주문 목록에서 주문번호를 클릭하면 상세 화면으로 이동합니다.

- **주문자 정보**: 이름, 전화, 이메일, 주소
- **품목 정보**: 제품명, 수량, 사이즈, 단가
- **결제 정보**: 결제방법, 금액, 입금자명
- **배송 정보**: 배송방법, 택배비, 송장번호

> **! 주의** 입금자명이 주문자명과 다를 경우 적색 배경으로 강조 표시됩니다.

### 2.4 교정 관리

고객이 업로드한 원고/시안 파일을 확인하고 관리합니다.

1. 사이드바 > 교정 관리 클릭
2. 주문번호별 파일 목록 확인
3. "보기" 버튼 클릭 → 이미지 뷰어로 원본 크기 확인
4. 필요 시 추가 파일 업로드 (드래그&드롭 지원)

지원 파일 형식: JPG, PNG, GIF, PDF, AI, PSD, ZIP (각 20MB 제한)

### 2.5 결제 현황

KG이니시스 카드결제 및 무통장입금 현황을 확인합니다.

- 결제 상태: 성공/실패/대기
- 결제 금액, 거래번호(TID), 결제일시
- 카드결제 완료 시 관리자 이메일 자동 알림

> **듀얼 도메인 결제 시스템**
>
> KG이니시스/KB에스크로는 두 도메인(dsp114.com + dsp114.co.kr) 공용으로 운영됩니다.
> - **KG이니시스**: MID `dsp1147479`, Sign Key `cEdnbCtISFZ1QUNpNm5hbG1JY1RlQT09` (두 도메인 공용)
> - **KB에스크로**: dsp114.com용 mHValue `eb30fbb0bc1da7fdcaf800c0bceebbff201111241043905` (롤백값 보존)
> - **SITE_URL 동적 감지**: 결제 returnUrl/closeUrl 자동 대응
> - 결제 시점에 접속 도메인에 맞춰 자동으로 적절한 URL로 리다이렉트됩니다.

### 2.6 택배 관리

택배 발송 정보를 관리합니다.

**배송 정보 입력 항목**

| 항목 | 설명 | 비고 |
| --- | --- | --- |
| 운임구분 | 착불/선불 선택 | 기본값: 착불 |
| 박스 수량 | 관리자 직접 입력 | 회사마다 박스 규격 다름 |
| 택배비 | 관리자 직접 입력 | 전화 확인 후 확정 |
| 송장번호 | 로젠택배 송장번호 | 발송 후 입력 |

> **! 주의** 무게는 자동 추정되지만, 박스수/택배비는 반드시 관리자가 직접 입력해야 합니다.

택배비 선불 확정 시 고객에게 이메일이 자동 발송되며, 고객은 마이페이지에서 결제할 수 있습니다.

## 3. 소통·견적

### 3.1 이메일 발송

회원에게 일괄 이메일을 발송합니다.

**이메일 작성 방법**

1. 사이드바 > 이메일 발송 클릭
2. 수신자 선택: 전체 회원 / 조건 필터 / 직접 입력
3. WYSIWYG 편집기로 본문 작성 (굵은글씨, 이미지 삽입, 링크 등)
4. 테스트 발송 → 본인 이메일로 확인
5. 본 발송 (100명씩 배치, 3초 간격)

> **! 주의** 네이버 SMTP 사용: 일일 약 500통 한도. Gmail 수신 시 스팸 분류 가능성 있음.

> **TIP** `{{name}}` 태그로 수신자 이름 자동 치환 가능

### 3.2 채팅 관리

영업시간(09:00~18:30) 직원 채팅 위젯과 야간 AI 챗봇이 자동 전환됩니다.

| 시간대 | 위젯 | 색상 |
| --- | --- | --- |
| 09:00~18:30 | 직원 채팅 (실시간 응대) | 주황색 |
| 18:30~09:00 | AI 챗봇 (자동 가격 조회) | 보라색 |

AI 챗봇은 9개 품목 가격 조회, 인쇄 가이드, 디자인비, 파일 제출 안내 등을 자동 응답합니다.

### 3.3 견적 관리

고객 견적서를 작성하고 이메일로 발송합니다.

**견적서 상태 흐름**

`draft(임시저장)` → `sent(발송됨)` → `viewed(고객 열람)` → `accepted/rejected(승인/거절)`

**견적번호 체계**

| 시스템 | 접두어 | 형식 | 예시 |
| --- | --- | --- | --- |
| 관리자 견적서 | AQ | AQ-YYYYMMDD-NNNN | AQ-20260208-0004 |
| 플로팅 견적받기 | FQ | FQ-YYYYMMDD-NNN | FQ-20260216-001 |

> **TIP** 견적서는 팝업 창으로 열리며, 콘텐츠 크기에 맞춰 자동 리사이즈됩니다.

견적 목록에서 개별/일괄 삭제가 가능합니다 (하드 삭제, 복구 불가).

### 3.4 고객 문의

홈페이지 문의게시판에서 접수된 문의 목록을 확인하고 답변합니다.

## 4. 제품·가격 관리

### 4.1 9개 품목 개요

| # | 품목명 | 폴더명 | 단위 | 가격 방식 |
| --- | --- | --- | --- | --- |
| 1 | 전단지 | inserted | 연 | DB 조회 |
| 2 | 스티커 | sticker_new | 매 | **수학 공식** |
| 3 | 자석스티커 | msticker | 매 | DB 조회 |
| 4 | 명함 | namecard | 매 | DB 조회 |
| 5 | 봉투 | envelope | 매 | DB 조회 |
| 6 | 포스터 | littleprint | 매 | DB 조회 |
| 7 | 상품권 | merchandisebond | 매 | DB 조회 |
| 8 | 카다록 | cadarok | 부 | DB 조회 |
| 9 | NCR양식지 | ncrflambeau | 권 | DB 조회 |

### 4.2 스티커 가격 계산 (특수)

스티커는 다른 8개 품목과 달리 DB 가격표가 아닌 **수학 공식**으로 가격을 계산합니다.

**입력 파라미터**: 재질(jong), 가로(garo), 세로(sero), 수량(mesu), 도무송(domusong), 디자인(uhyung)

**계산 흐름**:

재질코드 → 수량별 요율 조회 → 기본가격 → 도무송비 → 특수용지비 → 사이즈 마진 → 관리비 → 디자인비 = 공급가 → ×1.1 = VAT포함가

> **! 주의** 스티커 가격은 DB 조회가 아닌 수학 공식입니다. 혼동하지 마세요.

### 4.3 가격 관리

각 품목의 카테고리별 가격표를 수정합니다.

- 카테고리 추가/수정/삭제
- 수량별 단가 설정
- 스티커 요율 수정 (스티커수정 메뉴)

### 4.4 갤러리 관리

고객 주문 교정 이미지 갤러리를 관리합니다. 갤러리 샘플 + 실제 주문 이미지 혼합 표시 (24개/페이지).

## 5. 관리·통계

### 5.1 회원 관리

가입 회원 목록을 확인하고 관리합니다.

- 회원 목록 조회 / 검색
- 회원 정보 수정
- 사업자 정보 확인

비밀번호는 bcrypt 암호화 저장되며, 구 평문 비밀번호는 로그인 시 자동 업그레이드됩니다.

### 5.2 주문 통계

기간별/품목별 주문 통계를 확인합니다.

- 요약 카드 4개: 오늘 주문, 이번 달, 전체 누적, 평균 단가
- 기간 필터: 오늘/7일/30일/사용자 지정
- 품목별 매출 비율

> **TIP** 카드 숫자는 0에서 목표값까지 카운트업 애니메이션으로 표시됩니다.

### 5.3 방문자 분석

홈페이지 방문자 통계를 확인합니다.

- 일별/시간별 방문자 추이
- 인기 페이지 (영문 URL → 한글 품목명 자동 변환)
- 진입/이탈 페이지
- 실시간 방문자 테이블

### 5.4 사이트 설정

- 영문 버전 표시 ON/OFF 토글
- 기타 사이트 관련 설정

## 6. 업무 시나리오

### 6.1 일반 주문 처리 플로우

고객이 홈페이지에서 주문한 경우의 처리 절차입니다.

1. 고객 주문 → 대시보드 주문목록에 접수대기(0) 상태로 표시
2. 입금 확인 → 상태를 입금확인(2)으로 변경
3. 원고 확인 → 교정 관리에서 파일 확인
4. 시안 제작 → 교정 파일 업로드 → 시안확인중(4)
5. 고객 확인 → 인쇄준비(5) → 인쇄중(6) → 후가공(7) → 작업완료(8)
6. 택배 발송 → 송장번호 입력 → 발송완료(9)

### 6.2 택배 선불 주문 처리

1. 고객이 택배 선불로 주문
2. 주문완료 페이지: "택배비 확정 대기" + 결제버튼 비활성
3. 관리자: 주문 상세에서 택배비 입력 → 저장
4. 고객에게 택배비 확정 이메일 자동 발송
5. 고객: 마이페이지 → 주문상세 → 결제하기 (카드/무통장)

### 6.3 견적서 발송 플로우

1. 견적 관리 → 새 견적 작성
2. 품목/수량/금액 입력 → 저장 (draft)
3. 미리보기로 확인
4. "발송" 버튼 클릭 → 고객 이메일로 발송 (sent)
5. 고객 열람 → viewed → 승인/거절

### 6.4 전화 주문 등록

1. 전화로 주문 접수
2. 관리자 주문 메뉴에서 직접 등록
3. 품목/수량/가격 입력 + 주문자 정보 입력
4. 주문 등록 완료 → 주문 목록에서 확인

## 7. 주의사항 및 FAQ

### 7.1 일반 주의사항

- **브라우저 캐시 문제**: 화면이 이상하면 `Ctrl+Shift+Delete`로 캐시 삭제 후 새로고침
- **동시 접속**: 여러 탭에서 같은 주문을 수정하면 충돌 가능
- **세션 만료**: 8시간 미사용 시 자동 로그아웃

### 7.2 결제 관련

- **카드결제 테스트**: 로컬호스트에서는 반드시 테스트 모드로 사용
- 운영 서버(dsp114.com + dsp114.co.kr)에서만 실제 결제 처리
- 결제 실패 시 `payment/logs/` 폴더 확인
- **듀얼 도메인**: 두 도메인 모두 SITE_URL 동적 감지로 결제 정상 작동

> **! 주의** 테스트 시 소액(100~1,000원)으로 먼저 테스트하세요.
