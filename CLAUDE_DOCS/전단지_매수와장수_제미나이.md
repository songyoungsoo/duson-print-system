# 전단지 연/매수 표시 통일 시스템 (Gemini)

## 📌 문제 정의
- 주문 관리자, 주문 완료 페이지, 이메일 등 여러 곳에서 전단지 수량이 일관성 없이 표시되는 문제 발생.
- 예: 0.5연(4,000매) 주문이 '1개' 또는 '0.5연'으로만 표시되어 매수 정보가 누락되거나 잘못 표시됨.

## 🎯 해결 목표
- 전단지 품목의 수량을 **'연(ream)'**과 **'매(sheet)'**가 함께 표시되는 형식으로 통일.
- 예: "0.5연 (4,000매)"
- 주문 생성 시점부터 `quantity`(연), `unit`(단위), `mesu`(매수) 필드에 정확한 데이터가 저장되도록 하여 데이터 무결성 확보.

---

## 🛠️ 수정된 파일 및 내용

### 1. 주문 데이터 저장 로직 수정 (핵심 원인 해결)
- **파일:** `mlangorder_printauto/ProcessOrder_unified.php`
- **내용:**
    - 장바구니의 상품을 주문으로 변환하여 `mlangorder_printauto` 테이블에 저장할 때, 누락되었던 `quantity`(연 수)와 `unit`(단위: '연') 필드가 정상적으로 추가되도록 수정했습니다.
    - 이로 인해 전단지 주문의 핵심 데이터가 데이터베이스에 정확히 기록됩니다.

### 2. 관리자 주문 상세 보기 수정
- **파일:** `mlangorder_printauto/OrderFormOrderTree.php` (`admin.php`에 포함됨)
- **내용:**
    - 기존의 텍스트 방식 상세 정보를 제거하고, 주문 품목을 명확하게 보여주는 표(Table) 형식으로 변경했습니다.
    - '비고' 열을 추가하여 전단지의 경우 매수 `(예: 4,000매)`를 표시하도록 수정했습니다.
    - '수량' 열에는 연수(예: `0.5`)만 표시하고, '단위' 열에는 `연`을 표시하여 가독성을 높였습니다.

### 3. 주문 완료 페이지 수정
- **파일:** `mlangorder_printauto/OrderComplete_universal.php`
- **내용:**
    - 주문 처리 로직(`ProcessOrder_unified.php`) 수정의 영향으로, 이 페이지에서도 데이터베이스의 정확한 `quantity`와 `unit` 값을 읽어오게 됩니다.
    - 전단지의 경우, 수량 열에 `0.5연 (4,000매)`와 같이 연과 매수가 함께 표시되도록 최종 수정했습니다.

### 4. 이메일 발송 양식 수정
- **파일:** `mlangorder_printauto/send_order_email.php`
- **내용:**
    - 주문 완료 시 발송되는 이메일 본문의 주문 내역 표에 '비고' 열을 추가했습니다.
    - 전단지 품목일 경우, 이 '비고' 열에 매수 `(예: 4,000매)`가 표시되도록 로직을 추가했습니다.

## ✅ 기대 효과
- 주문 생성부터 확인, 이메일 발송까지 모든 과정에서 전단지 수량이 'X.X연 (YYYY매)' 형식으로 일관되게 표시됩니다.
- 데이터베이스에 수량 정보가 정확히 저장되어 향후 발생할 수 있는 데이터 불일치 문제를 예방합니다.
